# RDP session crash (mstsc.exe, printer driver issue)

## Context
Работа велась внутри корпоративной сети (доменная инфраструктура, доступ к системам через RDP, включая 1С).

Пользователи жаловались, что не могут нормально работать в 1С через RDP. При открытии сессии - закрывалась сама по себе.

## Problem
При подключении:
- RDP открывается
- но через короткое время падает
- любые действия в 1С выполнить невозможно

Сначала подумал, что проблема в самой 1С.

## Investigation
Начал с проверки логов Windows (Event Viewer).

Нашёл:
- Application Error (Event ID 1000)
- падает процесс `mstsc.exe`
- код ошибки: `0xc0000005`
<img width="1081" height="550" alt="1111" src="https://github.com/user-attachments/assets/004b8b38-7b60-463d-84fa-cd060dbadf08" />
По коду ошибки стало понятно, что это не просто сбой, а проблема на уровне доступа к памяти.

В ошибке фигурировал файл:
- `LMFX1N4Z.DLL`
<img width="1152" height="180" alt="22222" src="https://github.com/user-attachments/assets/234dd807-b971-4ebd-aa1d-b8c1f81a0706" />

Проверил, где он находится:
- папка `spool\drivers` драйвер печати

## Analysis
Стало понятно, что проблема не в 1С, а в самом RDP-клиенте.

Связал это с тем, что:
- в RDP используется проброс локальных устройств
- включая принтеры

Предположил, что проблема может быть в драйвере печати при редиректе.

## Solution №1
Нужно файл подключения открыть с помощью текствого редактора и попать в его конфиг (.rdp).
- Был параметр: redirectprinters:i:1
- Поменял на: redirectprinters:i:0

Данный параметр отвечает за проброс локальных принтеров в RDP-сессию. С учётом выявленной причины сбоя, изменение выполнено внутри конфига путём ручного редактирования файла .rdp через текстовый редактор (Блокнот), поскольку в корпоративной среде изменение параметров через графический интерфейс может быть ограничено правами доступа. После внесения изменений подключение стабилизировалось, удалённая сессия не завершается аварийно.

## Solution №2
На втором устройстве при попытке редактирования файла .rdp возникала ошибка повреждения файла, что не позволило применить изменение стандартным способом. В качестве обходного решения было выполнено:
- Локальное дублирование библиотеки LMFX1N4Z.DLL в рабочий путь запуска (.rdp-файла).

Это позволило избежать загрузки проблемного модуля из системного каталога и стабилизировать работу RDP-сессии.

## Result
После этого:
- RDP перестал падать
- 1С начала работать нормально

## Notes
Корневая причина - конфликт драйвера печати, загружаемого в процесс mstsc.exe.
